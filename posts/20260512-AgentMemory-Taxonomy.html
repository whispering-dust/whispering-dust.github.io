智能体记忆的现在与未来————Elastic 中国 AI 搜索技术大会观后感

本篇文章由 Hermes Agent 辅助完成

最近刷到了 Elastic 中国 AI 搜索技术大会上关于 Memory 讨论的视频，觉得里面的概括和理解非常深刻，受益匪浅。原演讲来自 Elastic 全球副总裁肖涵（原 Jina AI 创始人兼 CEO），这里结合自己的理解整理一下。

原文链接：https://mp.weixin.qq.com/s/9OMs6buLJrFa_ApEN57z1w

1 为什么需要记忆？ ​

要理解记忆的重要性，先得看看没有记忆的智能体有什么问题。演讲中总结了三大核心缺陷：

缺陷具体表现后果
遗漏Agent 记不住之前说过的事，比如你跟它说过 911 相关事件，下次聊到类似话题它完全不记得对话断裂，用户需要不断重复自己
失真记忆被扭曲，产生幻觉，把 A 事安到了 B 事上信息污染，用户信任被逐步蚕食
信任侵蚀前两个问题叠加，用户对 AI 的信任被长期消耗整个行业的节奏被记忆问题拖住

这三个问题听起来简单，但解决起来极其困难。因为记忆不是简单的"存下来"就行——它涉及表征（Representation）、提取（Retrieval）、更新（Update）、遗忘（Forgetting） 等一系列复杂的环节。

而且这里有一个非常关键的问题——对话是实时的，提取是有成本的。抽多了慢，抽少了漏，这个平衡极难把握。

2 记忆研究三种流派 ​

记忆该如何表征？不同的答案催生了完全不同的系统架构。目前主要有三大流派：

流派技术实现代表优点缺点
数据库派向量数据库、SQL、Key-Value。对话经 LLM 抽取 facts，存入向量库或知识图谱，检索时注入上下文Milvus, PingCAP, Chroma结构化程度高，查询效率好Schema 锁死灵活性，"一条记忆该长什么样"被固定
文件派Markdown、纯文本。Agent 读文件、写文件、不断累积小龙虾, MemSearch透明可编辑、可版本化文件会膨胀，需要 intelligent forgetting
模型派大模型权重或上下文本身。模型自决定记什么、忘什么、整合什么Letta, ChatGPT, EverMind AI零配置、自适应完全黑盒、不可审计、难以调试

目前主流观点认为混合架构是最优的工程实现，但这三个流派各有其哲学基础。数据库派更接近传统软件工程的思维——结构、查询、索引；文件派更看重透明度和可控性；模型派则信仰"大模型会自己搞定一切"。

3 共识框架 ​

尽管流派不同，但主流工作流已经有高度共识。今天看到的大部分智能体记忆框架基本都 follow 这个流程：

  1. 提取：从聊天记录中，通过大模型提取事实或记忆结构
  2. 转化/存储：将这些记忆结构转化为三种形态之一——向量（语义检索）、知识图谱（关系检索）、时序数据库（时序非常重要，因为需要选择性遗忘）
  3. 混合检索：基于时序数据库、知识图谱和向量库做融合检索
  4. 后处理：检索结果不能直接丢进上下文，还需要解决两个问题：
    • 冲突解决：当检测出两个记忆相互矛盾时，该以哪个为准？
    • 时间衰减：旧记忆的权重该如何下降？

当然并不是每一个框架都采用所有技术，有些只选做几个，有些则全链都做。

关键 insight：RAG 之所以不够用，是因为它本身没有状态、没有层级。做 embedding、做 keyword search 的时候，并不包含状态也不包含时间关系。它是一个非常扁平的召回模型，并不擅长做记忆。这已经是目前的共识，所以才有很多图数据库、时序数据库想介入进来一起把 Agent 记忆模型做好。

4 一个典型案例：小龙虾 ​

很多人用过小龙虾，它是典型的文件派代表。具体拆一下它的结构：

真相源是一个 memory/ 文件夹，里面按天组织 Markdown 日记。另外有一份 memory.md 主文件，存长期记忆（包括用户画像）；还有一份 soul.md，存它自己的"灵魂画像"，也就是它对自己是谁、自己怎么做事的内部表征。

从真相源派生两个索引：SQLite FTS5 做 BM25 关键词检索，LanceDB 做向量检索。在索引之上做混合检索，用 RRF（Reciprocal Rank Fusion） 融合排序、MMR（Maximal Marginal Relevance） 去重，再加时间衰减。对外暴露两个接口：memory_search 和 memory_get。

在此之上，它还模仿了人类的睡觉习惯：深度睡眠、浅度睡眠、REM（快速眼动）。设置了一套机制，保证不是每条记忆都能升级为长期记忆，而是通过阈值和判断条件来选择性地晋升。

这个设计哲学值得思考：记忆系统的核心不是"怎么存更多"，而是**"怎么判断什么值得记"**。

目前学术界和工业界已经建立了一些评测基准：

基准特点测什么

MABenchICLR 2026精确检索、测试时学习、长程理解、冲突解决

6 四个值得细想的 Insight ​

  1. 统一范式还没出现：肖总说他在做智能体记忆调研时发现，统一的范式并没有形成。有些人从仿生学角度看，有些人从自己的使用习惯和工作效率上看。这非常像 2017 年 Transformer 没出来之前的深度学习——CNN、RNN、LSTM 百花齐放，直到 Transformer 一统天下。智能体记忆现在就处在这个"春秋战国"时期。

  2. 纯文本派是躺平：不想构建任何模型和仿生学层级，基本靠 grep、keyword search。但这几年搜索发展太快——从 RAG 到 Deep Research——很多人已经疲惫到说："反正一切终究会被大模型吃掉，何必去奋斗？****" 但我个人认为这个想法有明显天花板。

  3. Unlearning——该还的技术债了：目前所有系统都是只增不减，这实际上是一笔技术债。2009 年做机器学习的时候就讨论过 Machine Unlearning：怎么把已经训练、已经学到的东西忘掉？一个现实类比："刷抖音这号废了，重新开号吧"。推荐系统一旦被污染就无法恢复，本质就是它不会遗忘。智能体记忆面临完全一样的问题——噪声一旦写入，就永久污染所有后续检索。

  4. 推荐系统会因为智能体记忆重新火起来：智能体记忆的核心是个性化：记住用户偏好、行为模式、历史决策。这和推荐系统做了二十年的事完全一样。小红书、抖音、淘宝这些推荐系统做得好的公司，凭借经验和先天优势，可能会在这个时代发挥出来。

7 未来 & 新路径 ​

今天我们讨论的记忆，不管是小龙虾还是别的，都假设 Agent 是一个一对一的个人助理，这是相对简单的场景。但把 Agent 丢进一个群聊里呢？它该记谁的？怎么分别记每个人的？记忆的未解决问题还非常多。

几个核心的开放问题：

  • 多方记忆隔离：群聊场景下，如何区分不同人的记忆并能分别检索？
  • 跨模型迁移：今天用智谱，明天换 MiniMax，记忆能不能无缝搬过去？
  • Unlearning 机制：如何实现真正的"遗忘"，而非简单地删除数据？
  • 统一表征：能不能有一个类似 Transformer 的统一范式出现，终结这个"春秋战国"时代？

目前工程实现上存入 Markdown 或结合数据库、知识图谱的居多，但多模态上的探索还非常有限。Markdown 显然不适合存储多模态的记忆（图片、音频、视频等）。

还有一个比较有意思的路径：将记忆抽象成图片，再用视觉模型来读取。DeepSeek 的 DeepSeek-OCR 系列模型就是这方面的典型代表。

补充：DeepSeek-OCR 核心方案

  • DeepSeek-OCR: Contexts Optical Compression (arXiv 2510.18234)
  • DeepSeek-OCR 2: Visual Causal Flow (arXiv 2601.20552)

DeepSeek-OCR（论文：DeepSeek-OCR: Contexts Optical Compression，arXiv:2510.18234）提出了一种"光学压缩"的思路——

核心架构由两部分组成：DeepEncoder（编码器）和 DeepSeek 3B-MoE-A570M（解码器）。核心思想是将长文本信息通过光学映射（2D 图像化）进行压缩，把文本"画"成一张图，然后用视觉模型从图中"读"回来。

关键发现：当文本 token 数量在视觉 token 数量的 10 倍以内（压缩比 < 10x）时，解码（OCR）精度可达 97%。即使在 20x 压缩比下，精度仍有约 60%。

实用价值也很强：在 OmniDocBench 上，只用 100 个视觉 token 就超越了 GOT-OCR2.0（256 token/页），用不到 800 个视觉 token 优于 MinerU2.0（6000+ token/页）。单张 A100-40G 每天可生成 20 万页以上 的训练数据。

DeepSeek-OCR 2（论文：Visual Causal Flow，arXiv:2601.20552）更进一步——提出了 DeepEncoder V2，赋予编码器因果推理能力。打破了传统 VLM 从左上到右下的固定光栅扫描顺序，让模型能够根据图像语义动态重排视觉 token 的顺序，更接近人类的视觉感知方式。它探索了一个新范式：能否通过两级联的 1D 因果推理结构来实现真正的 2D 图像理解？

这个思路对智能体记忆的启发：如果能把对话历史、用户画像等记忆内容"画"成一张结构化的图像，再用视觉模型去理解，或许能突破纯文本记忆在结构化存储和检索上的瓶颈。当然这还处于非常早期的探索阶段。
